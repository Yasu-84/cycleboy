# 競輪予想Webアプリ 開発要件書

## 1. システム概要

### 1.1 システム名

**競輪予想システム**

### 1.2 目的

競輪の車券購入に向けた予想分析を行い、当日の開催情報・レース情報を確認できるWebアプリケーション。
AI（LLM）を活用して、定義したポイントに基づく予想を自動生成する。

### 1.3 ターゲットユーザ

- 競輪初心者〜中級者
- データ分析が得意ではないが、予想を楽しみたいユーザ
- スマホ利用が中心

## 3. 機能

### 3.1 機能概要

- 定時(AM05:15)に起動するスクレイピングにより、以下の順序でデータを取得
  1.レース日程
  2.レースプログラム
  3.出走表(基本情報)
  4.出走表(直近成績)
  5.出走表(対戦表)
- スクレイピング完了後、AI予想ディスパッチャーがAI予想を自動生成
- スクレイピング失敗/AI予想の失敗に対応するため手動起動を可能とする
- 出走表一覧、出走表（基本情報、並び予想、直近成績、対戦成績、ＡＩ予想）を表示
- クライアントは認証なしで利用可能とする
- クライアントからのデータ更新は許可しない
- 数人の知人で使用するため、ログイン画面を設けないので認証機能は不要

### 3.2 スクレイピング

- サイト名: ネットケイリン
- ベースURL: `https://keirin.netkeiba.com/`

### 3.3 今後の拡張案

- 開催情報表示: 出走表一覧、出走表の表示機能
- AI予想: LLMによるAI予想
- AI予想表示:出走表(AI予想)の表示機能
- 予想履歴分析: 過去の予想精度の分析

## 4. データ

### 4.1 データ定義

1.レース日程.md
2.レースプログラム.md
3.出走表(基本情報).md
4.出走表(直近成績).md
5.出走表(対戦表).md

### 4.2 データ保存

- データの保持期間は7日
- 保持期間を経過したデータは、定時(AM05:00)に自動削除

## 5. 開発方針

- 段階的に実装を進めるため、今回の開発スクレイピング機能の実装のみとする。
- ジョブ機能と画面機能で共通で参照するデータの定義は共通化とする
- テーブル単位でデータのCRUDを実装して、ジョブ機能と画面機能で共通で使用する
